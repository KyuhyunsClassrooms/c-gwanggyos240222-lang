## 🐛 발견한 오류 #1

### 1) 어떤 문제가 발생했나요?
입력한 글자가 알파벳이 아닐 경우에도 그대로 처리되어 버립니다.
예를 들어, 숫자나 기호를 입력해도 guessed 배열에 추가돼요.

### 2) 오류가 발생한 코드
scanf(" %c", &guess);
guess = tolower(guess);

### 3) 왜 오류가 생겼나요?
문자 입력을 받을 때 입력값이 알파벳인지 검사하는 코드가 없어서 숫자, 특수문자, 공백 등이 그대로 등록됩니다.

### 4) 어떻게 고쳤나요?
scanf(" %c", &guess);
guess = tolower(guess);

if (!isalpha(guess)) {
    printf("알파벳만 입력할 수 있습니다.\n");
    continue;
}


### 5) 고친 후 결과

## 🐛 발견한 오류 #2

### 1) 어떤 문제가 발생했나요?
단어를 모두 맞췄을 때 프로그램이 바로 종료(return 0)되는데,
이러면 게임 종료 메시지 출력 이후 추가 코드 실행이 불가능해지고 구조가 깔끔하지 않습니다.

### 2) 오류가 발생한 코드
if (correct) {
    printf("\n축하합니다! 단어 '%s'를 맞추셨습니다!\n", game.word);
    return 0;
}

### 3) 왜 오류가 생겼나요?
반복문 안에서 바로 return 0을 사용하면 main 함수가 즉시 끝나기 때문에
추가 출력이나 종료 메시지 작성이 어렵고, 코드의 흐름이 갑자기 끊깁니다.
### 4) 어떻게 고쳤나요?
if (correct) {
    printf("\n축하합니다! 단어 '%s'를 맞추셨습니다!\n", game.word);
    break;  // return 대신 break 사용
}

### 5) 고친 후 결과
- [ ] ✅ 잘 작동해요!

## 🐛 발견한 오류 #3 (더 있으면 추가로 작성하세요)

### 1) 어떤 문제가 발생했나요?
guessed 배열의 최대 길이가 27자로 고정되어 있어 배열 범위를 넘어갈 가능성이 있습니다.

### 2) 오류가 발생한 코드
game.guessed[len] = guess;
game.guessed[len + 1] = '\0';

### 3) 왜 오류가 생겼나요?
알파벳은 26개이지만, 사용자가 중복 입력 없이 여러 번 오타를 입력하면 guessed 배열이 꽉 차서
배열 범위를 넘어 쓰기(overwrite)가 발생할 수 있습니다.
### 4) 어떻게 고쳤나요?
if (strlen(game.guessed) >= 26) {
    printf("더 이상 문자를 저장할 수 없습니다.\n");
    continue;
}

## 💭 디버깅하면서 배운 점

입력받은 문자가 알파벳인지 검사하는 것이 중요하다는 것을 알고, 배열을 할 때 크기 확인은 필수라는 것을 알았다.return을 함부로 사용했을 때 코드 구조가 복잡해질 수도 있고, 디버깅할 때 printf를 활용하면 문제 원인을 쉽게 찾을 수 있다는 것 또한 알 수 있었다. 비록 내가 100% 다 혼자서 코드를 짠 것은 아니지만 AI의 도움을 받아 무사히 코드를 짤 수 있었다.
